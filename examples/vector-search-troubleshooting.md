# Vector Search Troubleshooting

## Проблема: нерелевантные результаты (йод → сироп для собак)

### 1. Колонка `embedding_topic` заполнена?

Эмбеддинги должны быть **заранее вычислены** и сохранены в БД. Если колонка пустая или NULL — поиск вернёт мусор.

**Проверка:**
```sql
SELECT COUNT(*) as total, COUNT(embedding_topic) as with_embedding 
FROM smf_topics;
```

Если `with_embedding` = 0 или сильно меньше `total` — эмбеддинги не заполнены.

**Решение:** Нужен скрипт/триггер, который при создании/обновлении темы вызывает embeddings API и пишет вектор в `embedding_topic`. Модель и размерность должны совпадать с настройками MCP.

### 2. Совпадение модели и размерности

- Эмбеддинги в БД должны быть созданы **той же моделью**, что и `POSTGRES_MCP_EMBEDDINGS_MODEL` (qwen3-embedding-8b).
- Размерность вектора в колонке должна совпадать с `POSTGRES_MCP_VECTOR_DIMENSIONS` (2000).

### 3. Откуда брать текст для эмбеддинга

Обычно: `subject` + первые N символов `body` первого сообщения. Если эмбеддинг считали из пустого текста или "Re: Re: Re:" — вектор бесполезен.

### 4. Фильтр пустых сообщений

В системном промпте добавь `where_clause`:
```
where_clause: m.body IS NOT NULL AND m.body != ''
```

### 5. Альтернативная модель

qwen3-embedding-8b может плохо работать с узкими доменами (химия, медицина). Попробуй:
- `nomic-embed-text` (Ollama)
- `bge-m3` (многоязычный, 1024 dims — нужно менять VECTOR_DIMENSIONS и тип колонки)
